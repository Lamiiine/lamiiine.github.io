Turn On & Off Terminal Echoing

Aug 25, 2021

Echo is a feature that outputs any key pressed to the terminal and allows you to see what you are typing.Usually, this is useful in canonical mode; However, it could be a hassle if you are trying to hide the characters you are typing in the terminal. In this tutorial, I will guide you through the steps to help you enable and disable raw mode.

Setup: I won’t be using any external libraries so you only need a C compiler and the std library it comes with it. I will also be using the make program(this is not necessary, so don’t worry if you don’t have it). Check if you have a C compiler installed on your machine, type the following into you command line cc –version (cc stands for C Compiler). Also, if you want to check whether make is installed, type the following: make –version.

Turning off Echoing or canonical mode: By default, the terminal is set to run on canonical mode or cooked mode which reflects to the terminal every key press. I will be showing you how to turn-off & On echoing:

Turn Off Echoing: I will be using <termios.h>, which is the UNIX API for terminal.

#include <termios.h>
#include <unistd.h>
void turnOffEcho()
{
    struct termios raw;
    tcgetattr(STDIN_FILENO, &raw);
    raw.c_lflag &= ~(ECHO);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
}

int main()
{
    turnOffEcho();

    char c;
    while (read(STDIN_FILENO, &c, 1) == 1 && c != 'q')
        ;
    return 0;
}

tcgeattr(): it helps read terminal attributes to a struct. (https://pubs.opengroup.org/onlinepubs/7908799/xsh/tcgetattr.html) c_Iflag(): an interger local flag used to control terminal functions
ECHO: is flag defined as 4 bytes (32bits) tcsetattr(): it helps apply the changes to the terminal. (https://pubs.opengroup.org/onlinepubs/7908799/xsh/tcsetattr.html) TCSAFLUSH: it waits for all the output to be written to terminal to apply the changes. (https://pubs.opengroup.org/onlinepubs/7908799/xsh/tcflush.html)

Run the program: Go to the folder where the c file is located Run the terminal Type cc fileName.c -o fileName If no errors occur, an executable with fileName is produced. If so, type ./fileName You can type in the terminal and see raw mode in effect. To quit, you can type q then enter and this will disregard any input after the q.

Turn On Echoing: To turn things back into normal. We need to adjust the code so it turns on Echo as soon as we quit. To do so, we need to adjust the above code accordingly.

#include <stdlib.h>
#include <termios.h>
#include <unistd.h>
struct termios orig_termios;
void turnOnEcho() {
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &orig_termios);
}
void turnOffEcho() {
    tcgetattr(STDIN_FILENO, &orig_termios);
    atexit(turnOnEcho);
    struct termios raw = orig_termios;
    raw.c_lflag &= ~(ECHO);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
}
int main() {
    turnOffEcho();
    char c;
    while (read(STDIN_FILENO, &c, 1) == 1 && c != 'q')
        ;
    return 0;
}

We first create a global struct variable orig_termios to save the terminal’s original attributes and apply the changes to struct raw.

atexit(): a function that takes a function argument to be called when the program is terminated. (https://man7.org/linux/man-pages/man3/atexit.3.html)

Now you can compile this program and run it again to see the changes applied.

Bonus: If you get tired of having to write the command line to compile your program, you can save time and use make instead. How?

Create a Makefile using the terminal in the same folder of your c file. Copy the following into the terminal: fileName: fileName.c $(CC) fileName.c -o fileName -Wall -Wextra -pedantic -std=c99 fileName is what we are looking to build and it needs fileName.c to do so. (CC) a variable which translates exactly to cc (c compiler)

-Wall -Eextra and -pendantic returns warnings and std=c99 tells the machine which C language std we are using. Save the file then run make using terminal and your program will compile if no errors occur.
