Turn On & Off Terminal Echoing

Aug 25, 2021

Echo is a feature that outputs any key pressed to the terminal and allows you to see what you are typing.Usually, this is useful in canonical mode; However, it could be a hassle if you are trying to hide the characters you are typing in the terminal or building complex user interfaces. Therefore, in this writeup, I will guide you step by step to help you turn on & off Echoing.

Setup:

I won’t be using any external libraries so you only need a C compiler and the std library it comes with it. I will also be using the make program(this is not necessary, so don’t worry if you don’t have it installed).
First, check if you have a C compiler installed on your machine by typing the following into you command line

cc –version

(cc stands for C Compiler). And to check if make is installed, type the following:

make –version

Turning off Echoing or canonical mode:

By default, the terminal is set to run on canonical mode or cooked mode which reflects to the terminal every key press. I will be showing you how to turn-off & On echoing:
I will be using <termios.h>, which is the UNIX API for the terminal.

	#include <termios.h>
	#include <unistd.h>
	void turnOffEcho()
	{
	    struct termios raw;
	    tcgetattr(STDIN_FILENO, &raw);
	    raw.c_lflag &= ~(ECHO);
	    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
	}

	int main()
	{
	    turnOffEcho();

	    char c;
	    while (read(STDIN_FILENO, &c, 1) == 1 && c != 'q');
	    return 0;
	}

tcgeattr(): it helps read terminal attributes to a struct.
c_Iflag(): an interger local flag used to control terminal functions
ECHO: is flag defined as 4 bytes (32bits).
tcsetattr(): it helps apply the changes to the terminal.
TCSAFLUSH: it waits for all the output to be written to terminal to apply the changes.

Running the program:

  • Go to the folder where the c file is located
  • Run the terminal
  • Type cc fileName.c -o fileName
  • If no errors occur, an executable with fileName is produced. If so, type ./fileName
  • You can test the program by typing in the terminal and see raw mode in effect.
  • To quit, you need to type q then enter and this will disregard any input after the q.

Turn On Echoing:

To turn things back into normal. We need to adjust the code so it turns on Echo as soon as we quit. This requires a little adjustment to the code above.

	#include <stdlib.h>
	#include <termios.h>
	#include <unistd.h>
	struct termios orig_termios;
	void turnOnEcho() {
	    tcsetattr(STDIN_FILENO, TCSAFLUSH, &orig_termios);
	}
	
	void turnOffEcho() {
	    tcgetattr(STDIN_FILENO, &orig_termios);
	    atexit(turnOnEcho);
	    
	    struct termios raw = orig_termios;
	    raw.c_lflag &= ~(ECHO);
	    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
	}
	int main() {
	    turnOffEcho();
	    char c;
	    while (read(STDIN_FILENO, &c, 1) == 1 && c != 'q')
		;
	    return 0;
	}

We first create a global struct variable orig_termios to save the terminal’s original attributes and apply the changes to struct raw.

atexit(): a function that takes a function argument to be called when the program is terminated.

Now you can compile this program and run it again to see the changes applied.

Bonus:

If you get tired of having to write cc fileName.c -o fileName every time you want to compile your code, you can save time and use make instead. How?

  1. Create a Makefile using the terminal in the same folder of your c file.
  2. Copy the following into the terminal: fileName: fileName.c $(CC) fileName.c -o fileName -Wall -Wextra -pedantic -std=c99
  3. Type make into the terminal then hit enter.
  4. If no errors occur, type ./fileName to run your program.

fileName is what we are looking to build and it needs fileName.c to do so.
(CC) a variable which translates exactly to cc (c compiler).
-Wall -Wextra and -pendantic returns warnings and std=c99 tells the machine which C language std we are using.
